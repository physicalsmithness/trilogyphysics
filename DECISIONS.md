# Trilogy Physics: DECISIONS

Newest at the bottom. Cite decisions with their substance, never bare numbers (standing principle 4). Seed set dated 2026-06-08, written by the GCSE Physics Overview seat to stand the project up. The Architecture chat owns this file from here on and should ratify, amend, or supersede these as it engages the material; several are deliberately provisional.

---

**d001 (Trilogy is its own project, a peer of Pre-IB).** Trilogy Physics is a standalone project with its own Architecture chat, not a wing of Pre-IB / single science. The two cross-fertilise as peers. Sourced from Smith, 2026-06-08.

**d002 (Architecture and Housing combined; Authoring separate; split housing only on trigger).** One chat owns both project shape and the engine/display housing, plus one or more Authoring chats. Rationale: mirrors the deliberate Fields choice, which works, and keeping schema and display in one head is the cheaper default for a single-engine project. There is a known cross-borrowing risk when housing is split out (the IB electricity display could have borrowed more from the Fields driller), but Smith is not convinced that gap was caused by the split, so this is a default-of-convenience, not a claim about cause. Trigger to split off a dedicated Housing/Implementation chat: when engine work consistently starves authoring attention, or when a shared cross-board GCSE engine emerges that needs its own owner. Sourced from Smith, 2026-06-08.

**d003 (item schema: start from the Pre-IB v0.5 structure plus the Codex calculation vocabulary; authoring may evolve it).** Use the Pre-IB topic-7 schema v0.5 (multiple qtypes, atom registries) as the structural starting point, and the Codex six-layer calculation tagging vocabulary (shape, formula/relation/rule, number sources, unit handling, mark categories, ECF gates) as the source for calculation-item attributes. This is a starting reference, not a freeze; per standing principle 5, authoring proposes schema changes from the material. Provisional pending Architecture's M1 schema lock.

**d004 (atom-level dashboard ships in v1).** The per-atom coverage and per-misconception fire/avoid display is the differentiating feature and ships in v1, not deferred to a post-telemetry milestone (standing principle 1, and the Electric Circuits M9-deferral miss). The schema logs misconception_id and applicable-but-avoided from day one.

**d005 (tier is a first-class tag; cross-tier shared items tagged consistently).** Every item carries its tier (Foundation, Higher, or both). The same question often appears as a Foundation Q05/06 and a Higher Q01; tag both consistently, mirroring the Codex data which already does this. Standing principle 6.

**d006 (cohort-results stance: difficulty signal, not coverage driver).** Use the AQA national average mark per part as a signal of what is hard and what a hard question looks like, for calibration and sequencing. Do NOT author primarily against our own cohort's weak spots: the cohort does well partly because of existing drilling, so absence-of-drilling is not the target. Sourced from Smith, 2026-06-08.

**d007 (two-pass authoring: spec-first, then past-paper-mine, then gap).** Authors first think up questions that cover the spec point (so coverage is not narrowed to what has historically been asked), then mine the full past-paper bank for how it has really been examined, then close the gap with what has not been asked but plausibly could be. Go one way, then the other. Sourced from Smith, 2026-06-08.

**d008 (every author gets the full structured bank, not just the PDFs).** Authors are given the Codex DB (full extracted, tagged bank) plus past papers plus the question-type summary, not raw PDFs alone. Sourced from Smith, 2026-06-08 ("it would be stronger if they had a full bank").

**d009 (revive the existing repo in place; fold the Magnetism bank in).** The existing `trilogyphysics` Energy (thin) and Magnetism (about 160 questions) content is real work and is folded into the locked schema rather than rewritten from scratch. Whether the existing engine is kept, refactored, or replaced is an OPEN_QUESTION for Architecture/Housing.

**d010 (provisional home: this repo on G:; revisit on sync races).** Docs and code live here, co-located, the Fields pattern. If cloud-sync racing with active multi-chat edits causes trouble (the issue that moved Electric Circuits to a non-sync drive), move to a non-sync drive and update the Cowork binding. Provisional.

**d011 (mount the GCSE Physics Overview folder read-only in every Trilogy chat).** So standing_principles.md, SPEC_MAP.md, and packets stay reachable. This is the IB side's recurring blocker; institute from the start (standing principle 3).
